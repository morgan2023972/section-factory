# Release Notes - v1.3.0

Date: 2026-03-30

## Nouveautes

- Introduction de la validation AST-light dans la commande validate.
- Ajout d un routeur de regles progressif: off, advisory, warn, block.
- Ajout des parseurs AST-light dedies:
  - CSS: src/core/validation/parsers/cssAst.ts
  - JS: src/core/validation/parsers/jsAst.ts
  - HTML: src/core/validation/parsers/htmlAst.ts
- Integration du routeur AST-light dans le validateur principal via src/core/validation/ruleRouter.ts.

## CLI validate

Nouvelles options:

- --ast-validate
  - active la validation AST-light en mode advisory (non bloquant)
- --ast-phase <off|advisory|warn|block>
  - permet de piloter explicitement la phase AST

Comportement de rapport JSON:

- engine=regex-v1 quand AST-light est desactive
- engine=hybrid-v1 quand AST-light est active
- diagnostics AST ajoutes avec la source shopify-validator-ast-v1

## Compatibilite

- Aucun changement bloquant par defaut.
- Le mode par defaut reste off pour AST-light.
- Les comportements existants regex et strict/non-strict sont preserves.

## Qualite

- Ajout de tests unitaires pour:
  - parsing des options --ast-validate et --ast-phase
  - generation de rapport hybride avec diagnostics AST
  - verification d un flux advisory non bloquant dans sectionValidator
- Validation locale de la suite unitaire effectuee.

## Documentation

- README mis a jour avec les nouvelles options validate et exemples.
- ARCHITECTURE mis a jour avec la couche ruleRouter et parseurs AST-light.
- documentation-projet alignee sur les phases AST et le moteur hybride.
