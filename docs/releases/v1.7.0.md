## Changement principal

### Validator design system: robustesse + contrat d erreurs

Le validator expose desormais:

- `isValid` (etat global)
- `issues` (liste structuree avec `path` et `message`)
- `errors` (liste de messages maintenue pour compatibilite)

Le module accepte une entree `unknown` et ne throw plus sur les cas inattendus
(null, undefined, mauvais type, structures imbriquees non conformes).
