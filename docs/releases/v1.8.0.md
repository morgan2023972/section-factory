## Changement 1: designSystemValidator (recalibrage tests)

La suite est desormais strictement alignee au contrat actuel de
`validateDesignSystemCompliance(sectionCode: unknown)`:

- cas valide complet
- entree non-string (sans throw)
- absence de bloc `<style>`
- absence de `@media`
- absence de motion (`transition` / `animation` / `@keyframes`)
- absence de token CSS custom property (`--token`)
- absence de bouton scope sous `.section-{{ section.id }}`
- contrat de sortie (`isValid`, `issues`, `errors`)
- coherence `errors === issues.map(message)`

## Changement 2: sectionBuilder (nouveaux tests)

Nouveaux tests sur `writeSectionToDisk`:

- cas nominal complet avec nom de section complexe
- cas minimal valide + determinisme
- fallback sur `section.liquid` si nom sanitize vide
- erreurs sur code vide/whitespace/nullish
- verification chemin absolu de sortie
- absence de fuite de token brut `[object Object]` dans le nom de fichier
- robustesse minimale sur sectionName non-string (runtime)
