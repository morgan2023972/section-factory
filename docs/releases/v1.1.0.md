# Release Notes - v1.1.0

Date: 2026-03-19

Cette version pose une base solide pour fiabiliser la generation de sections Shopify, avec un focus sur la qualite, la testabilite et la collaboration equipe.

## Nouveautes

- Ajout de la commande CLI `--list-types` pour afficher les types de sections disponibles depuis le registre central.
- Ajout de la commande CLI `--list-sections` avec sortie enrichie (`type`, `alias`, `description`, `design-system`) et conservation de `--list-types` en compatibilite.
- Ajout de la commande CLI `--list-profiles` pour afficher les profils design disponibles avec un resume et le profil par defaut.
- Ajout de la commande CLI `validate` pour executer la validation sur un fichier section existant, sans generation IA.
- Ajout des modes `--strict` / `--non-strict` sur `validate` pour ajuster la severite de certaines regles.
- Rapport JSON `validate` versionne avec `reportVersion: 2` et `reportSchemaVersion: "1.1.0"`.
- Ajout d un mapping `ruleId` fin dans `validate` (schema, css, js, mobile, design_system) pour preparer l outillage et la migration AST.
- Ajout de la commande CLI `doctor` pour verifier l environnement (OPENAI_API_KEY, acces modele, dossiers output, version Node, fichiers de config).
- Ajout de l option `--format=json` sur `doctor` pour une sortie exploitable en CI/outillage.
- Mise en place d un registre central des types de sections comme source unique de verite.
- Ajout d un module de retry intelligent pour corriger automatiquement une section invalide via l IA.
- Integration du retry dans le pipeline de generation CLI avec support de `--max-retries`.

## Ameliorations

- Refactor du validator design pour utiliser le registre central au lieu d une liste locale.
- Renforcement de la structure CLI pour faciliter les tests d integration.
- Affichage CLI plus lisible pour la liste des types.

## Qualite et tests

- Ajout de tests unitaires pour:
  - mapping des types CLI
  - registre central des types
  - validation design
  - validation Shopify
- Ajout de tests d integration CLI incluant le scenario retry reussi apres un premier echec.
- Etat de validation: 63 tests passes, suite verte.

## CI/CD

- Ajout d un workflow CI minimal sur push et pull request.
- Pipeline CI: installation des dependances, tests unitaires, build.

## Documentation equipe

- Ajout d un README orienté equipe en francais.
- Ajout d un document d architecture.
- Ajout de templates de Pull Request et de message de commit.

## Impact

- Aucune rupture d API publique majeure.
- Version recommandee pour stabiliser les workflows de generation et validation avant extension fonctionnelle.
