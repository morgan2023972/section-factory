## Scenarios couverts

### sectionGenerator

- prompt vide reject
- OPENAI_API_KEY manquante
- fallback modele par defaut (`gpt-4.1-mini`)
- modele explicite via OPENAI_MODEL
- trim de output_text
- reponse OpenAI vide
- reponse OpenAI sans output_text
- erreur provider Error
- erreur provider non-Error

### retryGenerator

- construction du prompt de retry (type, tentative, regles, code precedent)
- fallback d issue quand la liste est vide
- succes au premier candidat valide
- succes apres plusieurs tentatives
- tentative avec reponse IA vide
- gestion des exceptions generateCorrection (Error et non-Error)
- normalisation des issues malformees
- validateCandidate sync et async
- fallback maxRetries invalide
- maxRetries non-entier (floor)
- seed initial currentCode/currentIssues quand originalCode/issues sont vides

### integration CLI generate

- `--max-retries` applique correctement dans le flux de correction
- echec apres retries epuises avec restitution des erreurs finales
- `--max-retries 0` sans retry (echec immediat)
